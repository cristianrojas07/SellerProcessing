# Documentación
El proyecto tiene que incluir:
- README.md con instrucciones paso a paso de cómo levantar todo
- Comandos para:
    - Levantar docker-compose
    - Aplicar migraciones
    - Correr cada proyecto
- URLs de acceso a cada componente
- Usuarios de prueba si implementás autenticación
La documentación tiene que ser lo suficientemente clara como para que alguien que
clona el repositorio pueda levantar el proyecto en menos de 10 minutos.
Importante: Todo tiene que estar en un monorepo (una sola solución .sln con todos
los proyectos adentro).
## Calidad de Código
- Nombres de variables y métodos en inglés
- Código limpio y legible
- Si usás algún patrón o decisión arquitectónica importante, comentala brevemente en el README
## Uso de inteligencia artificial
Alentamos su uso ya que es una herramienta que usamos a diario en el equipo. La
única condición: que puedas explicar y defender lo que entregás. Si usás IA, documentá en el README.md:
- Qué herramienta usaste (ChatGPT, Claude, Copilot, etc.)
- Para qué partes del challenge la usaste
- Cómo la usás normalmente en tu trabajo
Esto último nos interesa porque refleja cómo integrás estas herramientas a tu flujo de desarrollo
